Code style follows repo guidelines from AGENTS.md. Rust uses rustfmt, grouped imports by crate, snake_case modules, PascalCase types, and small functions with useful derives such as Debug/Serialize/Deserialize where appropriate. TypeScript/React uses ESLint + Prettier with 2-space indentation, single quotes, and ~80 columns; components use PascalCase, variables/functions use camelCase, practical kebab-case filenames. Generated files in `shared/types.ts` and `shared/remote-types.ts` must not be edited directly; regenerate from the Rust generator binaries instead. For frontend work, preserve existing design system/patterns unless explicitly changing them.
